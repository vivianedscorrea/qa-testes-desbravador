# Desafio Técnico de QA – Sistema de Reservas

## Objetivo

Este repositório contém a documentação e os cenários de teste elaborados para validar o fluxo de reservas do sistema disponibilizado pela Desbravador.

O objetivo do teste é identificar possíveis falhas, validar regras de negócio e garantir que o fluxo de reserva funcione corretamente.

---

## Sistema testado

Sistema de reservas online disponibilizado para o desafio técnico.

Fluxo principal testado:
- Busca de disponibilidade
- Seleção de quarto
- Inserção de dados dos hóspedes
- Processo de pagamento
- Confirmação da reserva

---

## Escopo dos testes

Foram elaborados cenários de teste baseados no fluxo principal do sistema, considerando:

- Cenários positivos (fluxo esperado)
- Cenários negativos (validações e erros)
- Regras de negócio relacionadas à reserva

---

## Estrutura do repositório

/docs → documentação dos testes  
/evidencias → prints da execução dos testes  
/cenarios → cenários escritos em BDD
/cypress → testes automatizados

---

## Ferramentas utilizadas

- GitHub para versionamento
- Markdown para documentação
- BDD (Gherkin) para escrita dos cenários
