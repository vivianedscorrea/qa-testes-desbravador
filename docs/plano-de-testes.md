# Plano de Testes

## Objetivo
Validar o fluxo de reserva do sistema de reservas online.

## Escopo
Serão testadas as seguintes funcionalidades:

- Busca de disponibilidade
- Seleção de quartos
- Inserção de dados dos hóspedes
- Processo de pagamento
- Confirmação da reserva

## Fora de escopo
- Integrações externas
- Performance do sistema

## Ambiente de testes
Sistema web de reservas disponibilizado para o desafio.

## Tipos de teste
- Testes funcionais
- Testes automatizados

## Ferramentas utilizadas
- GitHub (versionamento)

## Abordagem
- BDD (Behavior Driven Development)

## Linguagens
- TypeScript (automação de testes)
- Gherkin (definição de cenários de teste)
- Markdown (documentação)

## Framework de automação de testes
- Cypress

## IDE
- Visual Studio Code
